Procurement Agent Functionalities
=================================

This file documents the Procurement-side agents/components and what each one does.

1) AzureDIExtractorAgent
	- Purpose: Extract structured fields from uploaded procurement documents/forms.
	- Flow role: First extraction step for request prefill and form understanding.
	- Output: Header fields, confidence, and normalized extraction payload.

2) ComplianceAgent (Procurement)
	- Purpose: Validate recommendation/request against policy, standards, and compliance rules.
	- Flow role: Rule and control checkpoint before final recommendation acceptance.
	- Output: Violations, compliance status, and remediation guidance.

3) PerplexityMarketResearchAnalystAgent
	- Purpose: Collect external market intelligence for pricing, vendors, and trends.
	- Flow role: Primary market-research source.
	- Output: Market context, source-backed insights, and pricing references.

4) FallbackWebscraperAgent
	- Purpose: Provide market data when Perplexity is unavailable or insufficient.
	- Flow role: Fallback market-intelligence path.
	- Output: Supplemental web-derived snippets and comparable market references.

5) HVACRecommendationAgent (RecommendationAgent)
	- Purpose: Produce HVAC solution recommendation when deterministic rules are weak or no-match.
	- Flow role: Recommendation decision layer for procurement.
	- Output: Recommended option, confidence, tradeoffs, decision drivers.

6) ReasonSummaryAgent
	- Purpose: Convert recommendation output into reviewer-friendly reasoning summary.
	- Flow role: Human-readable explanation layer.
	- Output: Concise rationale, key reasons, and summary text for approvals/review.

7) RFQGenerator
	- Purpose: Generate RFQ document content from finalized request + recommendation context.
	- Flow role: Downstream document generation step.
	- Output: RFQ-ready structured content/document draft.

8) PlatformReasoningPlanner
	- Purpose: Decide which agents should run and in what order for a procurement request.
	- Flow role: Planning/orchestration decision point.
	- Output: Execution plan used by orchestration pipeline.

Current intended sequence (high-level)
--------------------------------------
Upload/Form -> AzureDIExtractorAgent -> ComplianceAgent checks ->
PerplexityMarketResearchAnalystAgent (or FallbackWebscraperAgent) ->
HVACRecommendationAgent -> ReasonSummaryAgent -> RFQGenerator

Note:
- FallbackWebscraperAgent runs only when primary Perplexity path fails or lacks coverage.
- PlatformReasoningPlanner controls sequencing logic across these steps.
