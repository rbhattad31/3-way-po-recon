Mermaid 2: Request-to-Run Sequence (Recommendation + Benchmark)

sequenceDiagram
  autonumber
  participant UI as UI (Workspace/Create)
  participant API as DRF ViewSet/Template View
  participant PERM as RBAC Permission Layer
  participant RS as RequestService
  participant AS as AnalysisRunService
  participant CT as Celery Task
  participant REC as RecommendationService
  participant BEN as BenchmarkService
  participant COM as ComplianceService
  participant DB as MySQL
  participant AUD as Audit/Trace

  UI->>API: Submit request fields + attributes
  API->>PERM: Check procurement.create
  PERM-->>API: allow
  API->>RS: create_request(payload)
  RS->>DB: Insert ProcurementRequest(status=DRAFT)
  RS->>DB: Upsert attributes
  RS->>AUD: log PROCUREMENT_REQUEST_CREATED

  UI->>API: Mark Ready
  API->>PERM: Check procurement.edit
  API->>RS: mark_ready(request_id)
  RS->>DB: validate required attributes
  RS->>DB: update status READY
  RS->>AUD: log status change

  UI->>API: Trigger run_type=RECOMMENDATION|BENCHMARK
  API->>PERM: Check procurement.run_analysis
  API->>AS: create_run(run_type, snapshot)
  AS->>DB: insert AnalysisRun(status=QUEUED,input_snapshot_json)
  API->>CT: run_analysis_task.delay(run_id)

  CT->>DB: load AnalysisRun + ProcurementRequest
  CT->>RS: update_status(PROCESSING)

  alt run_type == RECOMMENDATION
    CT->>REC: run_recommendation(request, run)
    REC->>AS: start_run(status=RUNNING)
    REC->>REC: deterministic _apply_rules(attrs)
    opt rules inconclusive and use_ai=true
      REC->>REC: call RecommendationAgent
    end
    REC->>COM: check_recommendation(final_result)
    REC->>DB: create RecommendationResult + ComplianceResult
    REC->>AS: complete_run(status=COMPLETED)
    REC->>RS: update request COMPLETED or REVIEW_REQUIRED
  else run_type == BENCHMARK
    CT->>BEN: run_benchmark(request, run, quotation)
    BEN->>AS: start_run(status=RUNNING)
    BEN->>BEN: resolve benchmark per line
    opt use_ai=true
      BEN->>BEN: call BenchmarkAgent when needed
    end
    BEN->>BEN: compute variance + classify risk
    BEN->>DB: create BenchmarkResult + BenchmarkResultLine
    BEN->>AS: complete_run(status=COMPLETED)
    BEN->>RS: update request COMPLETED or REVIEW_REQUIRED
  end

  CT->>AUD: observed_task + audit logs


Mermaid 3: Validation Deep Sequence (6 validators + optional agent augmentation)

sequenceDiagram
  autonumber
  participant UI as UI/API
  participant API as Endpoint validate
  participant PERM as RBAC
  participant AS as AnalysisRunService
  participant T as run_validation_task
  participant VO as ValidationOrchestrator
  participant RR as RuleResolver
  participant V1 as AttributeCompleteness
  participant V2 as DocumentCompleteness
  participant V3 as ScopeCoverage
  participant V4 as Ambiguity
  participant V5 as CommercialCompleteness
  participant V6 as ComplianceReadiness
  participant VA as ValidationAgent
  participant DB as MySQL

  UI->>API: POST validate(agent_enabled=true|false)
  API->>PERM: check procurement.validate
  API->>AS: create_run(run_type=VALIDATION,status=QUEUED)
  API->>T: run_validation_task.delay(run_id, agent_enabled)

  T->>VO: run_validation(request, run, agent_enabled)
  VO->>AS: start_run(RUNNING)
  VO->>RR: resolve_rules_for_request(domain_code,schema_code)
  RR-->>VO: active rules ordered by specificity + priority

  VO->>V1: validate(request,rules)
  VO->>V2: validate(request,rules)
  VO->>V3: validate(request,rules)
  VO->>V4: validate(request,rules)
  VO->>V5: validate(request,rules)
  VO->>V6: validate(request,rules)

  VO->>VO: aggregate findings, compute score, status, next action

  alt agent_enabled && ambiguous_count >= 3
    VO->>VA: augment_findings(request,run,findings)
    VA-->>VO: refined findings
  end

  VO->>DB: create ValidationResult
  VO->>DB: bulk create ValidationResultItem[]
  VO->>AS: complete_run(COMPLETED, confidence=score/100)
  T->>DB: map overall_status to request status READY/REVIEW_REQUIRED/FAILED

Mermaid 5: State Machines

stateDiagram-v2
  state "ProcurementRequest" as PR {
    [*] --> DRAFT
    DRAFT --> READY: mark_ready + required attrs valid
    READY --> PROCESSING: analysis or validation queued
    PROCESSING --> COMPLETED: compliance/risk/status acceptable
    PROCESSING --> REVIEW_REQUIRED: high risk or review needed
    PROCESSING --> FAILED: task/service exception or hard fail
    COMPLETED --> PROCESSING: rerun
    REVIEW_REQUIRED --> PROCESSING: rerun
  }

  state "AnalysisRun" as AR {
    [*] --> QUEUED
    QUEUED --> RUNNING: start_run
    RUNNING --> COMPLETED: complete_run
    RUNNING --> FAILED: fail_run
  }
