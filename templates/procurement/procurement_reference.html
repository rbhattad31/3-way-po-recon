Flow A — Product Selection Recommendation

Engineering-oriented flow. Given a store’s physical parameters and operational constraints, the platform recommends the most suitable HVAC system category, indicative capacity band, compliance alignment, and commercial constraints. The deterministic rule engine runs first; AI reasoning is only invoked when rules cannot reach a confident decision.

Flow B — Should-Cost Benchmarking

Procurement-oriented flow. Extracts supplier quotation lines from uploaded PDFs, classifies them into commercial categories, compares quoted rates against should-cost corridors, and produces a negotiation-focused commercial view with line-wise deviation analysis.

Document Extraction & Line Classification

Extraction Steps
  1. Upload & hash → store in Azure Blob Storage
  2. Azure Document Intelligence OCR → text + layout
  3. Structured extraction → document metadata + line items
  4. Each line: description / UOM / qty / unit rate / total / extraction confidence
  5. Low-confidence or missing commercial values → review flags
Line Categories
Equipment Controls Ducting Insulation Accessories Installation Testing & Commissioning

Raw descriptions are normalized and mapped to benchmark categories by the Line Classification Agent.

Benchmark Resolution Chain

1
Internal Catalogue Lookup Highest priority
Direct DB lookup against BenchmarkCatalogue or imported historical awarded rates. Returns full min/avg/max corridor. Confidence: 0.90+.
2
AI Benchmark Agent (BenchmarkAgent)
Routes through ProcurementAgentOrchestrator. AI estimates should-cost from known product family pricing corridors, OEM catalogue positioning, and geography factors. Confidence: 0.60–0.80.
3
Web Search Fallback (WebSearchService) Last resort
DuckDuckGo Instant Answer API first; Bing search scrape as secondary. Extracts price points from public text (AED/USD/SAR patterns). Always marked source="WEB_SEARCH". Confidence capped at 0.35. Requires manual validation.
4
No Benchmark Available
If all three sources return no data, line is marked VarianceStatus.WITHIN_RANGE with remark “No benchmark data available”. Flagged for manual review.

Audit & Governance

The procurement platform shares the same audit backbone as the parent platform. Every recommendation, benchmark run, agent invocation, and RBAC decision is logged as an AuditEvent record with full trace context and RBAC snapshot.

RBAC Roles

RoleProcurement Permissions

FINANCE_MANAGERFlow B benchmarking review, commercial approval, export
REVIEWEREngineering review of Flow A recommendations; sign-off on compliance
AP_PROCESSORCreate requests, upload quotations, run analyses — scoped to own submissions
AUDITORRead-only access to all records, governance timeline, and audit log

End-to-End Traceability

  • Every AnalysisRun carries a trace_id propagated through all service calls via TraceContext
  • Every RecommendationResult stores output_payload_json capturing the full rule engine + AI merge result
  • Every BenchmarkResultLine stores the benchmark source (source field: internal / AI / WEB_SEARCH) and confidence
  • Every agent invocation creates AgentRun + AgentStep + DecisionLog records with RBAC snapshot
  • Langfuse traces link every LLM generation to its parent AnalysisRun trace ID with per-flow scores
  • 100% of outputs are traceable to their source class: rules engine, AI reasoning, or external reference
