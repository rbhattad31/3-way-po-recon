When do agents run?
  • Intake / Extraction — Invoice Understanding agent classifies and validates the uploaded document.
  • PO Retrieval — PO Retrieval agent is the fallback when deterministic lookup fails.
  • GRN Analysis — GRN Retrieval agent investigates receipt discrepancies (3-Way only, conditional).
  • Exception Analysis — Exception Analysis + Reconciliation Assist agents reason over exceptions.
  • Review Routing — Review Routing agent assigns the case to the right reviewer queue.
  • Case Summary — Case Summary agent produces the final narrative for reviewers.
  • Reviewer Copilot — Interactive advisory assistant during human review (prompt-based, no autonomous run).

How Agents Choose a Recommendation
  • Each agent receives the reconciliation exceptions and uses tools to gather context.
  • The agent reasons through the exceptions and picks the most appropriate recommendation type.
  • A confidence score (0–100%) indicates how certain the agent is about its recommendation.
  • If confidence is below the threshold, the case is escalated for human review.
  • Multiple agents may run in sequence (e.g., Exception Analysis → Review Routing → Case Summary).
  • The Policy Engine may auto-close cases that fall within auto-close tolerance bands (qty: 5%, price: 3%, amount: 3%).
