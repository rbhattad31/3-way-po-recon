Recommendation Idempotency

Two layers prevent duplicates on retries or cross-run re-executions:

  1. Service-layer check (DecisionLogService.log_recommendation()) -- queries for an existing PENDING record with the same (reconciliation_result, recommendation_type) before inserting.
  2. DB unique constraint (uq_rec_result_type_run) -- enforces uniqueness on (reconciliation_result, recommendation_type, agent_run) as a last-resort safeguard against race conditions.

PostingMappingEngine

Resolution Strategy per Field

For each field (vendor, item, tax code, cost center, PO reference) the engine attempts resolution in this order. ERP connector is used when available (via ConnectorFactory).

  1. Exact code match against ERP reference table
  2. Alias lookup (VendorAliasMapping, ItemAliasMapping)
  3. Name / description fuzzy match
  4. Live ERP API connector (when connector= kwarg is provided)
Per-field resolution source (connector, fallback, confidence) is stored in PostingRun.erp_source_metadata_json for full provenance.

LLM Prompts

Three prompt categories — different roles:
Extraction
Used by InvoicePromptComposer to build the OCR-to-JSON system prompt. Modular: base + category overlay (goods/service/travel) + country overlay (GST/VAT). Single-shot call at temperature=0.0 — no tools, no ReAct loop.
Agent
Used by ReAct reasoning agents in the case pipeline (exception analysis, PO/GRN retrieval, reconciliation assist, etc.). Each agent has a dedicated agent.* slug. Runs tool-calling loops at temperature=0.1.
Case
Standalone prompts for case-level LLM calls: case summary, exception analysis narrative, non-PO validation reasoning, and the reviewer copilot assistant. Fetched via PromptRegistry by slug (e.g. case.case_summary).
All prompts are stored in the DB (PromptTemplate) and can be overridden via the Langfuse prompt editor or seed_prompts management command.

Agent-Level Tracing (AgentTraceService)

Single entry point for all agent activity. Produces: AgentRun → AgentStep → ToolCall / DecisionLog.

MethodRecord(s)What It Captures
start_agent_run()AgentRunReconciliationResult link, agent type, input payload, status=RUNNING
log_agent_step()AgentStepStep name, output, success flag, duration_ms
log_tool_call()ToolCallTool name, input/output JSON, success, duration_ms
log_agent_decision()DecisionLogDecision type, confidence score, evidence JSON
finish_agent_run()AgentRunstatus=COMPLETED/FAILED, confidence_score, summarized_reasoning

Span Hierarchy Example

HTTP POST /reconciliation/start/              <-- Root span (RequestTraceMiddleware)
  +-- @observed_action start_reconciliation         <-- Child (source_layer=UI)
       +-- @observed_service runner.run             <-- Child (source_layer=SERVICE)
            +-- AuditEvent: RECONCILIATION_STARTED  <-- Compliance audit record
       +-- @observed_task process_invoice_upload    <-- Celery header propagation
            +-- @observed_service orchestrator.run  <-- Child (source_layer=SERVICE)
                 +-- AgentTraceService.start_run()  <-- Agent-level tracing
