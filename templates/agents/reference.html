{# ── Three-path architecture callout ── #}
Three prompt paths — different builders, different callers:
Extraction (Path A — modular)
InvoicePromptComposer assembles: base + category overlay (goods / service / travel) + country overlay (GST / VAT). Composition result includes final_prompt, components dict (slug→version), and a 16-char prompt_hash for deterministic dedup. Single-shot LLM call at temperature=0.0 — no tools, no ReAct loop.
Governed Enrichment (step 1b, inside Path A — schema-driven v2.0)
Runs as step 1b of Path A via _run_governed_pipeline(), wrapped in try/except — if it raises any exception, Path A continues and labels the result Legacy source. Adds a governance audit trail only; does not drive the primary ExtractionResult.
PromptBuilderService (extraction_core/services/prompt_builder_service.py, PROMPT_VERSION = "2.0") builds prompts dynamically from ExtractionSchemaDefinition + TaxJurisdictionProfile — zero hardcoded country text. Assembles 7 sections: global rules → jurisdiction context → document type → schema fields → tax guidance → evidence/confidence rules → expected output schema. Supports hybrid mode via unresolved_field_keys. Persists prompt_code / prompt_version to ExtractionRun for audit.
Agent
ReAct reasoning agents in the case pipeline (exception analysis, PO/GRN retrieval, reconciliation assist, etc.). Each agent has a dedicated agent.* slug. Runs tool-calling loops at temperature=0.1.
Case Standalone prompts for case-level LLM calls: case summary, exception analysis, non-PO validation, reviewer copilot. Fetched via PromptRegistry by slug (e.g. case.case_summary).

  1. Langfuse — label production, 60 s in-process TTL cache. Source of truth in production.

  3. Hardcoded default — _DEFAULTS[slug] in apps/core/prompt_registry.py. Always present as ultimate fallback.
